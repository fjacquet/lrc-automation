    **Header:** `# ADR-007: Multiplatform Windows Support`
    **Status:** Accepted
    **Date:** 2026-03-06

    **Context section:** Describe that lrc-automation v0.6.0 extends from macOS-only to macOS + Windows. Lightroom Classic runs on both platforms. The existing codebase had four platform-specific gaps: catalog open (SQLite URI), process detection (pgrep), path writes (backslashes in SQL), and disk operations (AppleDouble cleanup). A cross-platform CI matrix was needed to verify correctness.

    **Decisions section** (document each as a named subsection):

    1. **SQLite URI Forward-Slash Fix (PATH-01)** — `_path_to_sqlite_uri()` in `catalog.py` converts the path to a POSIX string before building the `file:///` URI. Windows drive-letter paths like `C:\Users\...` become `file:///C:/Users/...`. Detection uses `path.as_posix().startswith("/")` to identify non-Windows paths (posix absolute paths start with `/`; Windows drive paths do not).

    2. **psutil for Process Detection (PROC-01)** — `psutil.process_iter(["name"])` replaces the `pgrep` subprocess call. psutil is a hard dependency with binary wheels for all target platforms. Checks `Adobe Lightroom Classic` (macOS) and `Lightroom.exe` (Windows). `AccessDenied` exceptions are caught and skipped.

    3. **path.as_posix() for SQL pathFromRoot Writes (PROC-02)** — All writes to `AgLibraryFile.pathFromRoot` use `path.as_posix()` so Lightroom (which expects forward slashes in its SQL schema) can locate folders after a move on Windows.

    4. **sys.platform == 'darwin' for AppleDouble Guard (PROC-03)** — AppleDouble (`._*`) file cleanup in `executor.py` is gated by `sys.platform == "darwin"`. This guard is identical to the existing pattern in the codebase and silently skips the step on Windows and Linux.

    5. **PermissionError Retry Loop (PROC-04)** — File moves in `executor.py` retry on `PermissionError` up to `_MOVE_RETRY_COUNT` times with `_MOVE_RETRY_SLEEP` seconds between attempts. Both constants are module-level so tests can zero them via monkeypatch. This handles transient antivirus scan locks on Windows.

    6. **SBOM via anchore/sbom-action (CI-04)** — `anchore/sbom-action@v0` generates an SPDX SBOM artifact attached to every GitHub release. Chosen over `actions/attest-sbom` because it produces a downloadable file visible on the releases page rather than a signed attestation.

    7. **.gitattributes LF Enforcement (CI-02)** — `* text=auto eol=lf` in `.gitattributes` prevents CRLF line-ending failures when `ruff format --check` runs on Windows CI runners with a fresh checkout.

    8. **Catalog Auto-Discovery (UX-01)** — `_discover_default_catalog(home_dir=None)` in `cli.py` finds the first `.lrcat` file in `~/Pictures/Lightroom/` (macOS/Linux) or `%USERPROFILE%\Pictures\Lightroom\` (Windows). The `--catalog` / `-c` option is now `required=False`. A `UsageError` guides users when discovery fails.

    **Consequences section:**
    - Positive: The tool now works on Windows for all core operations. CI matrix confirms correctness on 3 OS × 2 Python version combinations.
    - Negative: `reverse_geocoder` (the `[geo]` extra) has no Windows wheel on PyPI; location-folder features are macOS/Linux only.
    - Neutral: WAL mode was intentionally NOT enabled — Windows file-locking semantics prohibit it alongside the existing lock file strategy.
