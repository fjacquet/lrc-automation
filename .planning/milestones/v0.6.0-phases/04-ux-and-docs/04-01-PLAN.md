<objective>
Make `--catalog` / `-c` optional in the CLI by adding auto-discovery of the default Lightroom catalog path. When the user runs any subcommand without specifying `--catalog`, the tool checks `~/Pictures/Lightroom/` (macOS/Linux) or `%USERPROFILE%\Pictures\Lightroom\` (Windows) for the first `.lrcat` file and uses it automatically. If none is found, a clear `UsageError` explains how to specify one.

Purpose: Zero-config first run for users with a standard Lightroom installation.
Output: Updated `cli.py` with `_discover_default_catalog()` and updated `--catalog` option; new `tests/test_cli.py` with auto-discovery tests.
</objective>

  <behavior>
    - _discover_default_catalog(home_dir=None): when ~/Pictures/Lightroom/ contains Catalog.lrcat → returns str path
    - _discover_default_catalog(home_dir=None): when ~/Pictures/Lightroom/ does not exist → returns None
    - _discover_default_catalog(home_dir=None): when ~/Pictures/Lightroom/ exists but is empty → returns None
    - _discover_default_catalog(home_dir=tmp_path): when tmp_path/Pictures/Lightroom/Test.lrcat exists → returns str path to that file
    - cli() with no --catalog and auto-discover returns a path → proceeds (no "No catalog specified" error)
    - cli() with no --catalog and discovery returns None → raises UsageError with message containing "No catalog specified"
    - cli() with explicit --catalog path that does not exist → raises BadParameter containing "Catalog not found"
    - LRC_CATALOG_PATH env var → used as catalog, discovery not called
  </behavior>
  <implementation>
    Follow the pattern from the research file exactly:

    1. Add `import os`, `import sys` at top of cli.py (after existing imports).

    2. Add `_discover_default_catalog(home_dir: Path | None = None) -> str | None` function
       at module level, before `@click.group()`:
       - `base = home_dir or Path.home()`
       - For win32: `default_dir = base / "Pictures" / "Lightroom"`
       - For all others (darwin, linux): same path `base / "Pictures" / "Lightroom"`
       - If `not default_dir.is_dir()`: return None
       - `candidates = sorted(default_dir.glob("*.lrcat"))`
       - Return `str(candidates[0])` if candidates else None
       - If multiple candidates, also emit `console.print(f"[dim]Auto-selected catalog: {candidates[0]}[/dim]")`

    3. Change --catalog option:
       - `type=click.Path()` (drop `exists=True`)
       - `required=False`
       - `default=None`
       - Update help string to mention auto-discovery on macOS and Windows

    4. Update `cli()` body — add BEFORE the `catalog_path = Path(catalog)` line:
       ```python
       if catalog is None:
           catalog = _discover_default_catalog()
       if catalog is None:
           raise click.UsageError(
               "No catalog specified and none found at the default path.\n"
               "Use --catalog / -c or set LRC_CATALOG_PATH in your .env file."
           )
       catalog_path = Path(catalog)
       if not catalog_path.exists():
           raise click.BadParameter(
               f"Catalog not found: {catalog_path}", param_hint="--catalog"
           )
       ```
       Keep the existing `.lrcat` suffix check immediately after.
