<objective>
Fix the UX-03 packaging regression: reverse-geocoder appears in both core [project].dependencies and the [geo] optional extra (introduced in v0.5.0). Remove it from core dependencies and from the dev dependency group so that a plain `pip install lrc-automation` does not pull in the geo library.

Purpose: Any user who installs lrc-automation without wanting geocoding currently gets reverse-geocoder (a large C extension) pulled in automatically. This silently breaks on platforms where the wheel is unavailable and pollutes environments that do not need geo features.

Output: pyproject.toml with reverse-geocoder only in [geo] extra; tests/test_packaging.py confirming the import guard works without the library.
</objective>
