<objective>
Fix the path separator normalisation bug in scanner.py (PATH-02): three sites concatenate root_absolute_path (which may contain backslashes on Windows-created catalogs) directly with pathFromRoot before passing to extract_date_from_path(), which splits only on '/'. Backslash roots cause silent zero-date detection, making all Windows-catalog photos appear un-misplaced.

Purpose: Without this fix, scanning a catalog created natively on Windows returns 0 misplaced photos even when many photos are in wrong folders — a silent data-loss risk.

Output: Three one-line fixes using .replace("\\", "/") before string operations that split on '/'; new tests confirm correct behaviour with both forward-slash and backslash absolutePath values.
</objective>
