Replace the entire contents of `.github/workflows/release.yml` with the following:

```yaml
name: Release

on:
  push:
    tags:
      - "v*"

permissions:
  contents: write

jobs:
  release:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: astral-sh/setup-uv@v7
        with:
          python-version: "3.12"
          enable-cache: true

      - run: uv sync

      - run: uv run ruff check .
      - run: uv run ruff format --check .
      - run: uv run mypy src/
      - run: uv run pytest -v

      - name: Build package
        run: uv build

      - name: Generate SBOM
        uses: anchore/sbom-action@v0
        with:
          artifact-name: lrc-automation-sbom.spdx.json
          format: spdx-json

      - name: Create GitHub Release
        uses: softprops/action-gh-release@v2
        with:
          generate_release_notes: true
          files: |
            dist/*.whl
            dist/*.tar.gz
```

Key changes from original:

- `anchore/sbom-action@v0` with `artifact-name: lrc-automation-sbom.spdx.json` and `format: spdx-json`
- The existing `permissions: contents: write` already grants sbom-action the rights to upload release assets — no permission change needed
- SBOM step goes in this non-matrix release job only (not in ci.yml matrix) to avoid duplicate artifact name conflicts
- Do NOT use `actions/attest-sbom` — that creates a cryptographic attestation stored in GitHub's API, not a downloadable file on the releases page
