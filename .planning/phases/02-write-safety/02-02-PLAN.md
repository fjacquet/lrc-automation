<objective>
Fix three write-safety issues in `executor.py` and `reconciler.py`:
1. (PROC-02) Use `Path.as_posix()` when deriving `pathFromRoot` for SQL writes so forward slashes are always stored — `str(rel)` produces backslashes on Windows.
2. (PROC-03) Guard the AppleDouble `._*` check and deletion behind `sys.platform == "darwin"` so cleanup does not error or misbehave on Windows.
3. (PROC-04) Wrap the `shutil.move` / `os.rename` call in `_apply_file_op` with a 3-attempt retry loop that sleeps 0.5s on `PermissionError`, tolerating transient Windows antivirus scan locks.

Purpose: Without these fixes, Lightroom shows moved folders as missing on Windows (PROC-02), cleanup errors or deletes legitimate files (PROC-03), and a single AV lock causes a failed move with no recovery (PROC-04).
Output: Updated `executor.py`, `reconciler.py`, and new test functions in `tests/test_executor.py` and `tests/test_reconciler.py`.
</objective>

```python
def _is_effectively_empty(directory: Path) -> bool:
    """Return True if a directory contains no real files.

    On macOS, ``._<name>`` AppleDouble resource-fork files are skipped because
    they are metadata artefacts, not real content.  On other platforms ``._*``
    names are treated as real files.
    """
    for entry in directory.iterdir():
        if sys.platform == "darwin" and entry.name.startswith("._"):
            continue  # AppleDouble metadata — macOS only
        return False  # real file or subdirectory found
    return True
```
